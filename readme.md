# Portfolio

Here you can read about why I chose the tools that I did.

## HTML

All modern frameworks render down to HTML at the end anyway so I figured I'd
skip a step. For larger applications and sites a framework can help with
abstracting away a lot of the repetitive work however for the sake of having one
site maintained with a lack of using any frameworks I figured my portfolio would
be a good fit as it's fairly simple and doesn't need to be updated too
frequently except for the addition of blog posts and projects.

## Vanilla CSS

Vanilla CSS has come a long way and now has nesting, variables, new color
notations and plenty more features that near enough render preprocessers
obsolete. Yes these preprocessors can still contain some useful features however
they aren't the powerful improvement to CSS they once were more like a library
that adds some extra niceties.

## Web Components

React & Angular pioneered this huge wave of developers using components for
reusable pieces of code across the frontend, however Vanilla JavaScript actually
contains it's own api for creating web components and html as custom elements.
With these two things it is possible to recreate the reusable component
experience without the need to install any libraries or extra tooling
whatsoever.
